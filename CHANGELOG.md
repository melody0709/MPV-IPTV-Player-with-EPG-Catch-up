# Changelog
## V1.8.0 - 2026-04-02

- 新增本地私有覆盖配置机制：脚本会先读取默认配置，再读取本地私有覆盖配置，后者优先级最高，适合存放个人 EPG / M3U 地址
- 新增根目录 `release.py`：自动按当前版本号打包 zip，默认输出到 `release/`，并排除缓存、截图、`watch_later`、日志、已有 zip、`.git`、`.vscode`、AI 开发文档与本地私有配置
- 将 `script-opts/epg.conf` 调整为可发布的默认配置，个人订阅地址迁移到本地私有配置

## V1.7.8 - 2026-03-31

- 新增通用网络断线自动重试机制：播放失败时自动重试最多 3 次（每次间隔 1.5 秒），无需手动反复点击，彻底解决 `.php` 动态代理接口因上游响应慢或限流导致的 `Failed to open` 问题
- 新增 `file-loaded` 事件中的当前 URL 探测日志，便于调试流媒体连接状态
- 同步更新 `README.md`、`INSTRUCTIONS.md` 与脚本版本号为 V1.7.8

## V1.7.7 - 2026-03-31

- 修复 `find_channel_by_url` 冗余遍历：删除 `pairs(state.groups)` 兜底循环，`group_names` 与 `state.groups` 在 `parse_m3u` 中同步维护，无需二次遍历。
- 修复 `path:find` 误匹配（致命）：新增 `url_matches()` 函数，替换 5 处 `path:find(url, 1, true)`；只有当 URL 完全相等或其后紧跟 `?`/`#`/`/` 参数分隔符时才判定匹配，彻底解决 CCTV-11 被误认为 CCTV-1 的切台错位、历史记录存错、EPG 加载错乱问题。
- 性能优化：`path` 监听器从 O(N×M) 双层循环升级为 O(1) 字典查找；`parse_m3u` 解析时建立 `state.url_to_channel_map`（自动剥离 `?token=` 动态参数），HLS 切片切换不再卡顿。
- 性能优化：EPG 回看搜索菜单（F9）从全量渲染改为截断 100 条 + `on_search` 回调；Lua 内存搜索返回前 100 条，空查询自动恢复默认，菜单从卡顿数秒变为秒开。
- 性能优化：EPG 日期桶从相对 key 缓存升级为绝对日期预分桶（`state.epg_buckets[channel_id]["YYYY-MM-DD"]`）；解析时物理分桶，读取时 O(1) 按日期字符串提取，跨天零成本自动生效，无需定时器、无需缓存失效判断。
- 同步更新 `README.md`、`INSTRUCTIONS.md` 与脚本版本号为 V1.7.7。

## V1.7.6 - 2026-03-31

- 修复：避免在回看或菜单构建时因慢源/死链导致右键或 F8 菜单卡死 —— 将菜单中对播放进度的同步查询改为通过 `time-pos` 观察器缓存读取（`mp.get_property_number("time-pos")` 改为 `state.current_time_pos`）。
- 优化：将 `time-pos` 监听节流为秒级（`TIMEPOS_CHECK_INTERVAL` 默认为 1 秒），减少高频事件的无谓计算和 CPU 消耗。
- 优化：延迟生成回看 URL（菜单构建时不执行 `replace_catchup_time_params`，在用户点击时再生成），显著降低 F8/F9 打开菜单时的字符串替换负载。
- 修复：当播放源打开失败触发 HLS 兼容重试时，改为短延迟（约 0.35 秒）且可取消的重试，避免连续立即重试压缩 UI 响应窗口。
- 优化：在 `mpv.conf` 中启用 `network-timeout=5`，让坏源更快失败，减少主线程被阻塞的时长。
- 同步更新：`README.md`、`INSTRUCTIONS.md` 与脚本版本号为 V1.7.6。

## V1.7.5 - 2026-03-26

- 订阅模式启动优化为“缓存优先、后台刷新”：本地已有缓存时先秒开并恢复上次频道，再在后台刷新远程 M3U，减少直播首开等待
- 订阅模式历史记录改为稳定虚拟源标识，缓存文件刷新后仍可正确恢复上次频道
- `Shift+F9` 在订阅模式下升级为同时强制刷新 M3U 订阅与 EPG
- 修复 Windows 下订阅启动读取缓存时的 `subprocess args has incompatible type` 崩溃

## V1.7.0 - 2026-03-24

- 代码模块化重构：将 2488 行的单文件 `epg.lua` 拆分为 `scripts/epg/` 文件夹脚本，分为 5 个独立模块
  - `main.lua`（246 行）：入口文件，全局状态/选项初始化、所有 `mp.*` 事件绑定
  - `utils.lua`（449 行）：纯工具层，字符串/UTF8/OSD/时间日期/搜索拼音，无 state 依赖
  - `data.lua`（758 行）：数据层，EPG缓存/下载/解析、M3U解析、历史记录、频道查找
  - `playback.lua`（146 行）：播放控制层，URL加载/HLS重试/直播播放/回看参考时间/组内切台
  - `menu.lua`（572 行）：菜单层，4级菜单全部构建函数及EPG回看搜索菜单
- 模块间通过全局变量共享 `state`、`options`、缓存表，保持改动最小、行为不变
- 同步更新 `README.md`、`INSTRUCTIONS.md` 与脚本版本号
